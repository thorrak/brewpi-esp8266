Send a status message containing the current temperatures.

Can include optional annotations. The monitoring script will process and log this data. While this is also the response to the request for temperature ('t') command, it is used as a general purpose alert to the controller script. (This is why it is included in PiLink, and not CommandProcessor, despite the latter seeming to be a more logical location
